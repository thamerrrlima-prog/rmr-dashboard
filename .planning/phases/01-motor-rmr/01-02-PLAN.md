  <name>Task 1: Cálculo de R, M, Ritmo e GAP por cliente</name>
  <files>src/rmr_engine.py, tests/test_rmr_engine.py</files>
  <behavior>
    - compute_rmr(valid_df, reference_date=None) → pd.DataFrame indexado por ID com colunas: Recencia, Monetario, Ritmo, GAP
    - reference_date padrão: pd.Timestamp.today().normalize() (meia-noite)
    - Recência: (reference_date - data_ultima_compra).days — inteiro ≥ 0
    - Monetário: soma de Valor por cliente — float
    - Ritmo: se cliente tem N >= 2 compras, calcular datas únicas ordenadas, diffs consecutivos em dias, média dos diffs — float. Se N == 1: NaN
    - GAP = Ritmo - Recencia (NaN se Ritmo == NaN)
    - Cliente com apenas 1 compra: Ritmo = NaN, GAP = NaN — exclui do RMR
    - Test: cliente com 3 compras (diffs 10 e 20 dias) → Ritmo == 15.0
    - Test: cliente com 1 compra → Ritmo is NaN, GAP is NaN
    - Test: Recência de cliente cuja última compra foi reference_date → 0
    - Test: Monetário de cliente com 3 transações [100, 200, 50] → 350.0
    - Test: GAP = Ritmo - Recencia (ex.: Ritmo=15, Recencia=20 → GAP=-5)
  </behavior>
  <action>
    Em src/rmr_engine.py criar a função `compute_rmr(valid_df, reference_date=None)`:

    Parâmetros:
    - `valid_df`: DataFrame retornado por get_valid_transactions() — colunas canônicas: ID, Nome, PlayG, Telefone, Tipo, Valor, Data
    - `reference_date`: pd.Timestamp; padrão pd.Timestamp.today().normalize()

    Passos de implementação:
    1. Calcular Recência: agrupar por ID, pegar max(Data), subtrair de reference_date, converter para int (days). Coluna: Recencia.
    2. Calcular Monetário: groupby ID, sum(Valor). Coluna: Monetario.
    3. Calcular Ritmo:
       - groupby ID, ordenar Data, calcular diff entre datas consecutivas únicas (usar drop_duplicates na data antes de calcular diff para evitar mesma data contar duas vezes)
       - Se grupo tem >= 2 datas distintas: mean dos diffs em dias (float)
       - Se grupo tem apenas 1 data distinta: NaN
       - Coluna: Ritmo
    4. Calcular GAP: Ritmo - Recencia. Coluna: GAP.
    5. Manter colunas Nome, PlayG, Telefone no resultado (do último registro do cliente ou do join com Base B.U.)
    6. Retornar DataFrame com index = ID e colunas: [Nome, PlayG, Telefone, Recencia, Monetario, Ritmo, GAP]

    Nota: clientes com Ritmo=NaN permanecem no DataFrame retornado com GAP=NaN — a exclusão do RMR é filtrada no passo de scoring, não aqui. Isso permite ao Plan 03 distinguir clientes 1-compra.
  </action>
